### 步骤 5.2.1 创建服务入口
☑️ **操作**：新建 `ai-service/server.py`，完整代码如下：
```python
# ai-service/server.py
import base64
import logging
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel, Field
from typing import Optional
from ai_analyzer import ZhipuVisionAnalyzer, WhiteboardType, AIResponse

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger("AIServer")

app = FastAPI(title="云境智绘 AI 服务")

import os
API_KEY = os.environ.get("ZHIPU_API_KEY", "899b1cee727b455a8c1b99464223e395.Qpzgo6tGVrQzlHdd")
analyzer = ZhipuVisionAnalyzer(API_KEY)

class AnalyzeRequest(BaseModel):
    image_base64: str = Field(..., description="图片 Base64 编码（不含 data:image 前缀）")
    whiteboard_type: str = Field("general", description="内容类型：code, math, physics, general")
    context: Optional[str] = Field("", description="附加上下文，如课程名称")
    strict_format: Optional[bool] = Field(True, description="是否启用强制 JSON 输出")

class AnalyzeResponse(BaseModel):
    success: bool
    content: str
    error_msg: Optional[str] = None
    cost_time: float
    model_used: str

@app.post("/analyze", response_model=AnalyzeResponse)
async def analyze(request: AnalyzeRequest):
    try:
        type_map = {
            "code": WhiteboardType.CODE,
            "math": WhiteboardType.MATH,
            "physics": WhiteboardType.PHYSICS,
            "general": WhiteboardType.GENERAL
        }
        wb_type = type_map.get(request.whiteboard_type, WhiteboardType.GENERAL)
        response: AIResponse = analyzer.analyze_image(
            image_base64=request.image_base64,
            whiteboard_type=wb_type,
            context=request.context,
            strict_format=request.strict_format
        )
        return AnalyzeResponse(
            success=response.success,
            content=response.content,
            error_msg=response.error_msg,
            cost_time=response.cost_time,
            model_used=response.model_used
        )
    except Exception as e:
        logger.exception("AI 服务内部异常")
        raise HTTPException(status_code=500, detail=f"AI 服务内部错误：{str(e)}")

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```
**要点**：服务监听端口 8000；API Key 优先读环境变量，硬编码值仅用于本地开发，正式上线前必须删除。

### 步骤 5.3.4 创建 Controller
☑️ **操作**：新建 `backend/src/main/java/com/yourcompany/board/controller/AiController.java`。**注意**：若遇到 `@Value` 注入失败导致 404，可先使用硬编码的 AI 服务地址（如下所示）：
```java
package com.yourcompany.board.controller;

import com.yourcompany.board.dto.AnalyzeRequest;
import com.yourcompany.board.dto.AnalyzeResponse;
import org.springframework.http.*;
import org.springframework.web.bind.annotation.*;
import org.springframework.web.client.RestTemplate;

@RestController
@RequestMapping("/api")
public class AiController {

    // 硬编码 AI 服务地址（若 @Value 无法注入时可使用此方式快速跑通）
    private String aiServiceUrl = "http://localhost:8000/analyze";

    private final RestTemplate restTemplate = new RestTemplate();

    @PostMapping("/analyze")
    public ResponseEntity<AnalyzeResponse> analyze(@RequestBody AnalyzeRequest request) {
        try {
            HttpHeaders headers = new HttpHeaders();
            headers.setContentType(MediaType.APPLICATION_JSON);
            HttpEntity<AnalyzeRequest> entity = new HttpEntity<>(request, headers);

            ResponseEntity<AnalyzeResponse> aiResp = restTemplate.exchange(
                    aiServiceUrl,
                    HttpMethod.POST,
                    entity,
                    AnalyzeResponse.class
            );
            return ResponseEntity.ok(aiResp.getBody());
        } catch (Exception e) {
            AnalyzeResponse errorResp = new AnalyzeResponse();
            errorResp.setSuccess(false);
            errorResp.setErrorMsg("后端调用 AI 服务失败：" + e.getMessage());
            return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(errorResp);
        }
    }
}
```

| E1   | `pip install` 后提示 Gradio 版本冲突                         | 本地安装了更高版本的 Gradio，其依赖的 FastAPI 版本要求不同   | 忽略警告，本项目不使用 Gradio，不影响微服务运行              |

| E4   | 后端启动成功但 `/api/analyze` 返回 404，DEBUG 日志中 `RequestMappingHandlerMapping` 只有 3 个映射 | `AiController` 未被 Spring 注册为 Bean，通常因 `@Value` 注入配置失败导致整个 Bean 创建跳过 | 1. 检查文件包路径是否在 `com.yourcompany.board.controller` 下<br>2. 暂时将 `@Value` 改为硬编码 `private String aiServiceUrl = "http://localhost:8000/analyze";` |

| E6   | 前端画布上只有移动黑点，无法留下连续笔迹                     | MediaPipe `onResults` 回调频繁执行 `clearRect`，覆盖了鼠标绘制 | 临时禁用 MediaPipe 绘制循环：将 `onResults` 置空，不调用 `camera.start()`，用鼠标模式接管画布 |
| E7   | 浏览器控制台报 `Tracking Prevention blocked access to storage` 或 `still waiting on run dependencies: wasm-instantiate` | CDN 隐私限制或 WebAssembly 加载失败                          | 1. 尝试强制刷新 (`Ctrl+Shift+R`) 或切换 Chrome<br>2. 优先使用鼠标模式完成闭环，手势优化留到后续阶段 |

# 开发纠结记录

1. **`@Value` 注入 vs 硬编码**  
   初期想通过配置文件管理 AI 地址，但 `@Value` 注入失败导致整个 Controller 静默跳过，无任何显式错误，排查耗时。结论：在第一阶段骨架搭建时，可优先硬编码快速跑通，配置化留到后期统一处理。

2. **Java/Python 字段命名风格冲突**  
   前端 JSON 已使用下划线 `image_base64`，但 Java DTO 默认序列化为驼峰，Python Pydantic 模型要求完全匹配。没想到 Jackson 默认不会自动转换，直到看到 422 错误才意识到需要 `@JsonProperty` 显式映射。这提醒跨语言协作务必提前约定字段名。

3. **MediaPipe 手势加载卡住**  
   原本设计依靠手势隔空写字，但浏览器隐私策略或 CDN 不稳定导致 WebAssembly 一直等待依赖，画不出线。最后果断启用鼠标调试模式，先验证核心链路，证明“正方体”逻辑完全可用，手势问题降为后续专项优化。这让自己明白：原型阶段不要在一个非核心依赖上死磕，有替代方案就果断切换。

4. **404 排错耗时过多**  
   后端明明启动成功却总是 404，没注意到 `RequestMappingHandlerMapping` 的映射数量只有 3 个。直到开启 DEBUG 日志才发现 Controller 根本没注册。以后排查 404 应首先检查 Bean 映射日志，而不是盲目怀疑路径写错。
